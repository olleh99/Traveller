## 🛠 구현된 주요 기능

### 1. 백엔드 시스템 🔧

#### Socket.io 서버 구성
- **파일**: `backend/config/socket.js`
- **기능**: WebSocket 서버 관리, 실시간 통신, 사용자 연결 관리
- **주요 이벤트**: 
  - 연결/해제 관리
  - 알림 읽음 처리
  - 실시간 알림 전송

#### 알림 데이터 모델 
- **파일**: `backend/models/sequelize/Notification.js`
- **테이블**: `Notifications` 
- **필드**: 
  - 알림 유형 (매칭/메시지/리뷰/시스템 등)
  - 제목, 메시지, 관련 데이터
  - 읽음 상태, 읽은 시간
  - 메타데이터 (JSON)

#### 알림 API 엔드포인트
- **파일**: `backend/controllers/notificationController.js`, `backend/routes/notifications.js`
- **엔드포인트**:
  - `GET /api/notifications` - 알림 목록 (페이지네이션)
  - `GET /api/notifications/unread-count` - 읽지 않은 알림 수
  - `PUT /api/notifications/read` - 알림 읽음 처리
  - `PUT /api/notifications/read-all` - 모든 알림 읽음
  - `DELETE /api/notifications/:id` - 알림 삭제
  - `GET/PUT /api/notifications/settings` - 알림 설정 관리

#### 이벤트 트리거 시스템
**매칭 관련 알림** (`matchingController.js`):
- 매칭 요청 수신 알림
- 매칭 승인 알림  
- 매칭 거절 알림

**메시지 관련 알림** (`messageController.js`):
- 새 메시지 도착 알림

**리뷰 관련 알림** (확장 가능):
- 리뷰 도움됨 알림
- 리뷰 댓글 알림

## 📊 지원하는 알림 유형

### 현재 구현된 알림 (8가지)
1. **매칭 요청** (`matching_request`) - 새로운 동행 요청 수신
2. **매칭 승인** (`matching_accepted`) - 내 요청이 승인됨
3. **매칭 거절** (`matching_rejected`) - 내 요청이 거절됨  
4. **매칭 종료** (`matching_ended`) - 매칭이 종료됨
5. **새 메시지** (`new_message`) - 1:1 채팅 메시지 수신
6. **리뷰 도움됨** (`review_like`) - 내 리뷰가 도움됨 표시
7. **계획 공유** (`plan_shared`) - 여행 계획 공유됨
8. **시스템 알림** (`system`) - 공지사항 및 시스템 메시지

### 확장 가능한 알림 유형
- `destination_update` - 즐겨찾기 여행지 업데이트
- 사용자 정의 알림 유형 추가 가능

## 🔧 기술적 구현 세부사항

### 데이터베이스 스키마
```sql
CREATE TABLE `Notifications` (
  `id` INTEGER AUTO_INCREMENT PRIMARY KEY,
  `user_id` INTEGER NOT NULL,
  `type` ENUM(...) NOT NULL,
  `title` VARCHAR(255) NOT NULL,
  `message` TEXT NOT NULL,
  `relatedType` ENUM(...),
  `relatedId` INTEGER,
  `actionUrl` VARCHAR(255),
  `isRead` TINYINT(1) DEFAULT FALSE,
  `readAt` DATETIME,
  `metadata` JSON,
  `createdAt` DATETIME NOT NULL,
  `updatedAt` DATETIME NOT NULL,
  INDEX `user_id_isRead` (`user_id`, `isRead`),
  INDEX `user_id_type` (`user_id`, `type`),
  FOREIGN KEY (`user_id`) REFERENCES `Users` (`user_id`)
);
```

### WebSocket 이벤트 구조
```javascript
// 서버 → 클라이언트 이벤트
'notification:new' // 새 알림 전송
'notification:read:success' // 읽음 처리 성공
'notifications:readAll:success' // 전체 읽음 성공

// 클라이언트 → 서버 이벤트  
'notification:read' // 알림 읽음 처리 요청
'notifications:readAll' // 전체 읽음 요청
```
