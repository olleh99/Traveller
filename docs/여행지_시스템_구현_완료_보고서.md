#### 1.1 데이터베이스 모델 및 데이터
- **여행지 모델**: `backend/models/sequelize/Destination.js`
- **샘플 데이터**: 6개 여행지 (남산타워, 제주 우도, 부산 해운대, 경주 불국사, 전주 한옥마을, 여수 엑스포)

- **좌표 데이터**: 모든 여행지에 위도/경도 정보 추가

#### 1.2 백엔드 API
- **파일**: `backend/controllers/destinationController.js`
- **구현된 엔드포인트**:
  ```
  GET /api/destinations              - 여행지 목록 조회 (검색/필터링 지원)
  GET /api/destinations/:id          - 여행지 상세 조회
  POST /api/destinations             - 여행지 등록 (관리자)
  PUT /api/destinations/:id          - 여행지 수정 (관리자)
  DELETE /api/destinations/:id       - 여행지 삭제 (관리자)
  ```

#### 1.3 검색 및 필터링
- **검색 기능**: 이름, 설명, 주소로 통합 검색
- **카테고리 필터**: 관광지, 음식점, 숙박시설, 쇼핑, 문화시설, 기타

- **실시간 검색**: 500ms 디바운싱 적용

## 🐛 해결한 주요 오류들

### 1일차 오류들
1. **API 응답 구조 오류**
   **문제**: `response.data.destinations` vs `response.destinations`
   **해결**: axios 인터셉터 구조 파악 후 올바른 경로 사용

2. **amenities 데이터 타입 오류**
   **문제**: JSON 배열을 문자열로 처리하려 해서 `split()` 오류
   **해결**: 타입 체크 후 배열/문자열 각각 처리하는 로직 구현

3. **즐겨찾기 외래키 제약조건 오류**
   **문제**: `temporary-user-id`가 실제 DB에 존재하지 않음
   **해결**: 실제 존재하는 사용자 ID 사용

4. **MySQL GROUP BY 오류**
   **문제**: `sql_mode=only_full_group_by` 모드에서 집계 쿼리 오류
   **해결**: 리뷰 통계를 별도 쿼리로 분리

5. **리뷰 정렬 컬럼 오류**
   **문제**: 'created' 컬럼명 오류
   **해결**: 'created_at'로 수정

### 2일차 오류들 ⭐ NEW
6. **이미지 CORS 오류**
   **문제**: `net::ERR_BLOCKED_BY_RESPONSE.NotSameOrigin` - 다른 포트 간 이미지 로딩 차단
   **해결**: Helmet 설정에서 `crossOriginResourcePolicy: "cross-origin"` 추가, 정적 파일 CORS 헤더 설정

7. **ActivityDashboard 최근 활동 조회 실패**
   **문제**: `Cannot read properties of undefined (reading 'activities')`
   **해결**: API 응답 구조 검증 로직 추가, 안전한 데이터 접근 방식 적용

8. **MyReviewsView 라우터 등록 누락**
   **문제**: 네비게이션 클릭 시 네트워크 요청 없음
   **해결**: Vue Router에 `/my-reviews` 경로 추가 등록

9. **프로필 통계 API 응답 구조 오류**
   **문제**: `profileStatistics.value = response.data`에서 `undefined` 설정
   **해결**: API 응답 구조 확인 후 올바른 데이터 경로 설정

10. **여행계획 통계 숫자 표시 오류**
    **문제**: `GROUP BY`와 `findAndCountAll` 사용 시 `[object Object]` 표시
    **해결**: `planStatusStats` 객체 값들을 합산하여 총 개수 계산

### 3일차 오류들 ⭐ NEW
11. **디자인 통일화 진행 중 상태 관리 오류**
    **문제**: FavoritesView와 TravelPlansView의 통계 데이터 관리 방식 차이
    **해결**: 동일한 패턴으로 통계 계산 로직 구현, computed 속성 활용

## 🔧 기술적 아키텍처

### 백엔드 아키텍처
```
┌─────────────────┐
│   Routes        │ ← RESTful API 엔드포인트
├─────────────────┤
│   Controllers   │ ← 비즈니스 로직 처리
├─────────────────┤
│   Models        │ ← Sequelize ORM 모델
├─────────────────┤
│   Database      │ ← MySQL 데이터베이스
└─────────────────┘
```

### 프론트엔드 아키텍처
```
┌─────────────────┐
│   Views         │ ← 페이지 컴포넌트
├─────────────────┤
│   Components    │ ← 재사용 가능 컴포넌트
├─────────────────┤
│   API Clients   │ ← HTTP 통신 레이어
├─────────────────┤
│   Stores        │ ← 상태 관리 (Pinia)
└─────────────────┘
```

### 데이터 흐름
```
User Input → Vue Component → API Client → Backend Controller → Database
                ↓
User Interface ← Vue Reactive State ← API Response ← JSON Response ←
```
