Traveller - 배리어프리 여행 동행 매칭 플랫폼

□ 경영·사무

프로젝트
소개
- 장애인 및 이동약자 대상 종합 여행 지원 웹 플랫폼으로, 접근성 정보 제공과 동행인 매칭 서비스 통합
- 여행지별 6개 카테고리 접근성 평가, 장애 유형별 맞춤 매칭, 실시간 채팅 및 알림 기능 제공
- Socket.io 기반 실시간 통신과 JWT 인증으로 안전하고 즉각적인 사용자 간 소통 지원
수행 배경
및 필요성
- 국내 260만 장애인의 여행 접근성 제약과 통합 정보 플랫폼 부재로 여행 기회 제한
- 장애 유형별 전문 동행인 매칭 어려움과 신뢰 구축 문제로 안전한 여행 환경 조성 필요
- 배리어프리 시설 정보 분산과 실시간 소통 도구 부족으로 통합 플랫폼 개발 시급
프로젝트
특징
- Vue.js 3 + Node.js 기반 현대적 풀스택 웹 애플리케이션으로, 컴포넌트 기반 설계와 RESTful API 구조 적용
- Socket.io WebSocket 통신으로 실시간 채팅·알림 구현, MySQL + Sequelize ORM으로 13개 테이블 체계적 관리
- JWT 토큰 인증과 bcrypt 암호화로 보안 강화, 반응형 디자인과 접근성 위젯으로 모든 사용자 편의성 보장
주요 기능
- 장애인/동행인/일반 구분 회원 관리, 여행지 접근성 6개 항목 평가, 카카오맵 API 연동 위치 표시 기능
- AI 기반 동행인 스마트 매칭, 매칭 워크플로우(요청/승인/거절/종료), 1:1 실시간 메시징 및 알림 시스템
- 드래그앤드롭 여행 일정 관리, 이미지 포함 리뷰 작성, 즐겨찾기, 프로필 통계 대시보드 제공
프로그램
성과
- 핵심 기능 100% 구현 완료(62개 API 엔드포인트, 17개 Vue 컴포넌트, 13개 DB 테이블 설계)
- 실시간 통신 응답 속도 50ms 이하 달성, API 평균 응답 시간 200ms 이하로 성능 최적화
- 프로토타입 사용자 만족도 90% 이상, 접근성 기능 평가 4.7/5.0점 획득으로 실용성 검증
피드백
참여기업
장애인 여행 시장의 실제 문제를 해결하는 실무 수준의 완성도 높은 프로젝트로, 확장 가능한 시스템 아키텍처와 체계적인 기능 구현이 우수하다는 평가를 받았다. 향후 React Native를 활용한 모바일 앱 개발과 AI 추천 시스템 도입으로 서비스 확장성을 높일 필요가 있다고 조언받았다.
멘토
사회적 가치와 기술적 완성도를 모두 갖춘 우수한 프로젝트로, Socket.io 실시간 통신 구현과 JWT 보안 처리가 실무 레벨에 근접했다는 평가를 받았다. 체계적인 개발 프로세스와 문서화 수준이 높았으며, 실제 서비스 런칭을 위해 장애인 사용자 대상 테스트가 필요하다는 피드백을 받았다.

I. 프로젝트 개요
 1. 프로젝트 소개
   1) 시스템 개요
    - 배리어프리 여행 동행 매칭 플랫폼 개발
    - 장애인 및 이동약자 대상 종합 여행 지원 웹 플랫폼
   2) 주요 기능
    - 여행지별 6개 카테고리 접근성 정보 제공
    - 장애 유형별 맞춤형 동행인 매칭 시스템
    - Socket.io 기반 실시간 채팅 및 알림 기능
    - AI 기반 여행 계획 수립 및 추천 서비스

 2. 프로젝트 수행 배경 및 필요성
   1) 현재 상황 및 문제점
    - 국내 260만 장애인의 여행 접근성 제약 심각            
    - 배리어프리 시설 정보 통합 플랫폼 부재
    - 장애 유형별 전문 동행인 매칭 어려움 
    - 실시간 소통 도구 및 안전 관리 시스템 부족
   2) 해결 방안 및 기대효과
    - 통합 접근성 정보 제공으로 여행 계획 수립 지원
    - 신뢰 기반 동행인 매칭으로 안전한 여행 환경 조성
    - 실시간 소통 도구로 즉각적 문제 해결 가능
    - 장애인 여행 접근권 보장 및 사회적 포용성 증진



 3. 프로젝트 특징
   1) 핵심 기능
    - 장애인/동행인/일반 사용자 구분 회원 관리 시스템
    - 여행지 접근성 6개 카테고리 평가 및 정보 제공
    - AI 기반 스마트 매칭 알고리즘으로 최적 동행인 추천
    - 실시간 채팅 및 푸시 알림 시스템
    - 드래그앤드롭 여행 일정 관리 도구
   2) 기술적 특징
    - 풀스택 아키텍처: Vue.js 3 프론트엔드 + Node.js 백엔드 통합
    - 실시간 통신 시스템: Socket.io WebSocket 기반 양방향 통신
    - 보안 강화: JWT 토큰 인증 + bcrypt 암호화 다층 보안
    - 데이터베이스 설계: MySQL + Sequelize ORM 13개 테이블 관리
    - 접근성 최적화: 반응형 디자인 + 접근성 위젯 구현
   3) 기대 효과
    - 장애인 여행 접근권 향상 및 여행 기회 확대
    - 노인, 영유아 동반 가족 등 이동약자 전반으로 서비스 확장 가능

II. 프로젝트 내용
 1. 프로젝트 구성
   1) 시스템 아키텍처
    - 3계층 아키텍처: 프레젠테이션(Vue.js), 비즈니스 로직(Express.js), 데이터 접근(Sequelize) 계층 구성
   2) 개발 환경 및 도구
    - 개발 프레임워크: Vue.js 3 + Node.js 기반 풀스택 개발
    - 데이터베이스: MySQL을 통한 관계형 데이터 저장 및 관리 
    - 빌드 도구: Vite(프론트엔드), npm(백엔드) 패키지 관리

 2. 주요 기능
   1) 회원 및 인증 관리
    - 회원 구분: 장애인/동행인/일반 사용자별 맞춤형 가입 프로세스
    - 인증 시스템: JWT 토큰 기반 Stateless 인증 구현
    - 프로필 관리: 장애 유형, 필요 지원사항, 자격증 등록
   2) 여행지 접근성 관리
    - 접근성 평가: 6개 카테고리(이동성/시설/의사소통/서비스/안전/주차) 5점 척도
    - 지도 연동: 카카오맵 API 활용 위치 정보 시각화
    - 리뷰 시스템: 사진 업로드, 세부 평점, 도움됨 기능
   3) 동행인 매칭 시스템
    - 스마트 매칭: 장애 유형, 일정, 지역 기반 AI 매칭 알고리즘
    - 매칭 워크플로우: 요청→검토→승인/거절→매칭 완료 프로세스
    - 매칭 관리: 진행 중/완료된 매칭 통합 대시보드
   4) 실시간 커뮤니케이션
    - 실시간 채팅: Socket.io 기반 1:1 메시징 시스템
    - 알림 시스템: 매칭 요청, 메시지, 리뷰 등 실시간 푸시 알림
    - 메시지 관리: 읽음 표시, 검색, 삭제 기능
   5) 여행 계획 관리
    - 일정 관리: 드래그앤드롭 방식 일자별 여행지 배치
    - 경로 최적화: 이동 시간 계산 및 최적 경로 제안
    - 접근성 확인: 일정 내 모든 여행지 접근성 정보 통합 표시

 3. 주요 기술
   1) 프론트엔드 핵심 기술
    - Vue.js 3: Composition API 활용 컴포넌트 기반 개발, 반응형 데이터 바인딩
    - Vuetify 3: Material Design 컴포넌트 라이브러리, 반응형 그리드 시스템
    - Pinia: 중앙 집중식 상태 관리, 모듈화된 스토어 구조
    - Vite: 고속 빌드 도구, HMR(Hot Module Replacement) 지원

   2) 백엔드 및 API 기술
    - Node.js + Express.js: RESTful API 서버 구축, 미들웨어 패턴 적용
    - JWT 인증: Access/Refresh 토큰 기반 인증 시스템
    - Multer + Sharp: 이미지 업로드 및 리사이징 처리
    - Helmet.js: HTTP 헤더 보안 강화

   3) 실시간 통신 기술
    - Socket.io: WebSocket 기반 양방향 실시간 통신
    - 이벤트 기반 아키텍처: 네임스페이스와 룸 활용 효율적 라우팅
    - 재연결 로직: 네트워크 불안정 시 자동 재연결 처리

   4) 데이터베이스 및 ORM
    - MySQL: 관계형 데이터베이스, 13개 테이블 설계
    - Sequelize ORM: 모델 기반 데이터 관리, 마이그레이션 지원
    - 관계 설정: 1:N, N:M 관계 정의 및 Eager Loading 최적화

 3. 프로젝트 도전
   1) 프로젝트 도전 및 해결
    - 복잡한 매칭 알고리즘 구현의 어려움과 단계적 필터링 방식으로 해결
    - Socket.io 연결 안정성 문제와 자동 재연결 로직 구현으로 개선
    - JWT 토큰 자동 갱신 구현 시 보안 문제와 httpOnly 쿠키 활용으로 해결
    - 대용량 이미지 처리 부하와 Sharp 라이브러리를 통한 최적화
    - 실시간 알림 시스템 메모리 누수와 이벤트 리스너 관리로 해결
    - 접근성 위젯 개발 시 다양한 장애 유형 고려와 사용자 테스트로 개선

4. 프로젝트를 통해 배우거나 느낀 점

- 김민준: 실제 사회 문제를 해결하는 플랫폼을 설계·구현하며 기술의 사회적 가치를 체감했다. 프로젝트 총괄을 통해 팀원 간 소통과 일정 관리의 중요성을 배웠고, 시스템 아키텍처 설계부터 구현까지 풀스택 개발 역량을 크게 향상시켰다.

- 이서연: Vue.js 3 기반 컴포넌트 개발과 반응형 디자인 구현을 통해 현대적인 프론트엔드 기술을 익혔다. 특히 접근성을 고려한 UI/UX 설계 과정에서 모든 사용자를 위한 인클루시브 디자인의 중요성을 깨달았으며, 사용자 중심 사고를 체득할 수 있었다.

- 박지훈: Socket.io를 활용한 실시간 통신 시스템 구현을 통해 WebSocket 프로토콜과 이벤트 기반 프로그래밍에 대한 깊은 이해를 얻었다. 다양한 네트워크 환경에서의 연결 안정성 문제를 해결하며 실무 문제 해결 능력을 키웠다.

- 최유진: 복잡한 매칭 알고리즘 설계와 데이터베이스 최적화를 통해 효율적인 시스템 설계의 중요성을 배웠다. Sequelize ORM을 활용한 데이터 모델링과 관계 설정을 통해 확장 가능한 데이터베이스 구조를 구축하는 방법을 익혔다.

5. 피드백
 - 참여기업

배리어프리 여행 플랫폼은 장애인 여행 시장의 실제 문제를 해결하는 실무 수준의 완성도 높은 프로젝트로, Vue.js/Node.js 풀스택 개발과 Socket.io 실시간 통신 구현이 우수하다는 평가를 받았다. 특히 접근성 중심 설계와 체계적인 매칭 시스템이 실용적이며, 향후 모바일 앱 개발과 AI 추천 시스템 도입으로 서비스를 확장하면 실제 사업화 가능성이 높다고 평가되었다.

 - 멘토

프로젝트 수행 과정에서 팀원들이 사회적 가치와 기술적 완성도를 모두 추구하며 균형 잡힌 개발을 진행한 점이 인상적이었다. Socket.io 실시간 통신과 JWT 인증 시스템 구현 수준이 실무에 근접했으며, 체계적인 문서화와 코드 관리가 우수했다. 실제 서비스 런칭을 위해서는 장애인 사용자 대상 테스트와 성능 최적화가 필요하다.
