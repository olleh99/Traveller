프로젝트
소개
- AI 기반 캡스톤디자인 관리시스템으로 대학생 프로젝트 수행을 지원하는 웹 플랫폼
- 학생이 선택한 기자재에 맞는 프로젝트 추천, 구입처 및 비용, 개발 방법 안내 기능 제공
- LLM API(Claude) 기반 맞춤형 질문 응답 및 개발 과정의 기술적 문제 실시간 해결 지원
수행 배경
및 필요성
- 캡스톤디자인에서 실무 역량 걍화와 기자재 및 주제 선정의 어려움, 기술적  지원 부족이 문제로 대두됨
- LLM을 활용한 맞춤형 프로젝트 추천과 기술 지원 필요
- 이를 통한 캡스톤디자인의 질적 향상과 효율적 학습 환경 조성 기대
프로젝트
특징
- AI 및 LLM 기반 캡스톤디자인 관리시스템으로, 학생이 선택한 기자재에 맞춰 프로젝트 추천 및 실시간 기술 지원 제공
- 기자재 정보 통합 관리, 팀 협업 도구, 진행 관리, 대시보드 등 다양한 기능 웹 플랫폼에서 지원
-  Claude API 활용  Q&A, 코드 얘시 등 맞춤형 기술 지원으로 캡스톤디자인 프로젝트의 효율성 제고 
주요 기능
- AI 기반 프로젝트 추천, 기자재 정보·대여·재고 실시간 관리, LLM 챗봇 상담, 실시간 채팅 등 협업 및 기술 지원 기능 제공
- 사용자 인증/권한 관리, 장비 대여·예약·수리, 커뮤니티(공지사항, 문의게시판, 자료실), 개인정보 및 활동 로그 관리 지원
- 관리자 대시보드를 통해 시스템 전체 현황 및 운영 상태를 한눈에 모니터링 가능
프로그램
성과
- 대학 및 교육기관의 캡스톤디자인 운영 효율성을 높이고, AI 기반 기술 문제 해결 능력과 실무 역량을 강화
- 프로젝트 관리 및 협업 시스템 구축 경험을 통해 향후 유사 학습관리시스템(LMS) 구축 레퍼런스 모델 활용 가능
- 통합 대시보드 등 결과물을 통해 실질적인 학습성과와 업무 생산성 향상 기대
피드백
참여기업
AI 기반 캡스톤디자인 관리시스템은 수업의 실무 활용도를 크게 높일 수 있으며, 맞춤형 기술 지원과 실시간 협업 기능을 통해 학습 및 과제 수행에 실질적인 도움을 줄 수 있다는 긍정적인 평가를 받았다. 또한, 개발 결과물은 향후 유사 플랫폼 개발이나 기업 교육용 플랫폼 모델에 참고할 가치가 높은 것으로 평가되었다.
멘토
프로젝트 수행 과정에서 학생들이 실무와 연계된 시스템 구현 경험을 통해 문제 해결 능력을 키우고, 협업과 소통 역량을 효과적으로 향상시킨 점이 인상적이었다. 기술 활용과 책임감 있는 태도 역시 우수했으며, 개발 결과물의 완성도도 높았다.


결과 보고서 본문


I. 프로젝트 개요
 1. 프로젝트 소개
   1) 시스템 개요
    - AI 활용 캡스톤 디자인 관리시스템 개발
    - 대학생 캡스톤 디자인 프로젝트 수행 지원용 웹 플랫폼
   2) 주요 기능
    - 학생 선택 기자재에 맞는 프로젝트 추천 기능 제공
    - 기자재 구입처, 비용, 개발 방법 안내
    - LLM API(Claude API) 활용 맞춤형 질문 응답 제공
    - 개발 과정의 기술적 문제 실시간 해결 지원

 2. 프로젝트 수행 배경 및 필요성
   1) 현재 상황 및 문제점
    - 캡스톤 디자인의 실무 역량 강화 중요성 증가            
    - 프로젝트 주제 선정 및 기자재 파악 어려움
    - 기자재 선택 후 진행 가능한 프로젝트 정보 부족 
    - 기술적 문제 발생 시 즉각적 도움 어려움
   2) 해결 방안 및 기대효과
    - LLM을 활용해 맞춤형 추천 및 기술 지원 가능
    - 캡스톤 디자인 질적 향상 및 효율적 학습 환경 조성 기대 

 3. 프로젝트 특징
   1) 핵심 기능
    - 프로젝트 생성, 작업 분담, 일정 관리 기능 제공
    - 기자재 대여/반납 및 실시간 재고 확인 기능 구현
    - AI 기반 기술 질문 응답 시스템 구축
    - 팀 채팅, 공지, 파일 공유 등 협업 도구 통합
    - 프로젝트 결과물 아카이브 및 통계 대시보드 제공
   2) 기술적 특징
    - AI 기반 기능형 업무 지원: Claude API를 활용한 능동적 프로젝트 보조 기능
    - 실시간 통신 아키텍쳐: WebSocket 기반 실시간 채팅 시스템 구현
    - 모듈화된 설계: 독립적 모듈 구성으로 유지보수성과 확장성 확보
    - 강화된 보안 시스템: Spring Security. 기반 다층 보안 구조
    - 통합 로깅 시스템: 사용자 및 관리 활동 추적하는 포괄적 로그 관리
   3) 기대 효과
    - 대학 및 교육기관의 캡스톤디자인 수업 운영 효율성 향상
    - 향후 유사 학습관리시스템(LMS) 구축의 레퍼런스 모델로 활용

II. 프로젝트 내용
 1. 프로젝트 구성
   1) 시스템 아키텍처
    - 3계층 아키텍처: 프레젠테이션(JSP), 비즈니스 로직(Spring Service), 데이터 	접근(MyBatis) 계층 구성
   2) 개발 환경 및 도구
    - 개발 프레임워크: Spring Framework 기반 MVC 패턴 적용
    - 데이터베이스: MariaDB를 통한 데이터 저장 및 관리 
    - 빌드 도구: Maven을 통한 프로젝트 의존성 관리

 2. 주요 기능
   1) AI 기반 업무 지원
    - 프로젝트 어시스턴트: Claude API 활용한 프로젝트 관련 질의응답 시스템
    - 실시간 챗봇: 업무 관련 문의사항에 대한 즉시 응답 가능
    - 대화 이력 관리: 사용자 모든 대화 내용 저장 및 검색 기능
   2) 실시간 커뮤니케이션
    - 그룹 채팅: 프로젝트별 복수명 단위의 그룹 채팅 기능
    - 개인 메시지: 1:1 개인 메시지 송수신
    - 파일 공유: 채팅 중 파일 첨부 및 공유 기능
    - 알림 설정: 채팅 알림 개인화 설정
    - 메시지 검색: 과거 대화 내용 검색 및 필터링
   3) 장비 관리 시스템
    - 장비 대여 관리: 신청, 승인, 반납 전 과정 관리
    - 장비 수리 시스템: 수리 신청부터 완료까지 추적 관리 
    - 장비 예약: 사전 예약을 통한 체계적 관리 및 운영
    - 특별 접근 권한: 고가/특수 장비에 대한 별도 승인 프로세스
   4) 프로젝트 협업 관리
    - 프로젝트 생성 및 관리: 프로젝트 기본 정보 설정 및 상태 관리
    - 팀원 초대: 프로젝트별 팀원 관리 및 권한 부여
    - 진행 상황 추적: 프로젝트 단계별 진행 상황 모니터링
    - 문서 사용권 및 권한 관리: 문서별 접근 권한 설정
   5) 사용자 관리
    - 계정 관리: 외부/내부 이메일 기반 계정 생성
    - 권한 관리: 일반 사용자, 관리자 등 역할 기반 접근 제어
    - 개인정보 관리: 마이페이지를 통한 개인정보 수정 및 비밀번호 변경 

 3. 주요 기술
   1) 백엔드 핵심 기술
    - Spring Framework: MVC 패턴 기반 웹 애플리케이션 프레임 워크, 의존성 주입과 AOP 활용
    - Spring Security: 인증/인가 처리, CSRF 방어 및 세션 관리
    - MyBatis: SQL 매핑 프레임워크, 복잡한 쿼리 처리 및 동적 SQL 생성
    - Maven: 빌드 및 종속성 관리 자동화 도구

   2)  AI 및 실시간 통신 기술
    - Claude API 연동: 프로젝트 주제에 따른 AI 질의응답 기능 구현
    - RESTful API: 비동기 방식의 클라이언트-서버 간 데이터 송수신 구조

   3) 프론트엔드 기술
    - JSP + Bootstrap: 템플릿 기반 사용자 인터페이스 구현, 반응형 디자인 적용
    - jQuery: 비동기 처리 및 사용자 이벤트 제어
    - JavaScript: 커스터마이징 된 UI 로직 구현

   4) 데이터베이스 및 설정 관리
    - XML 기반 설정: Spring 설정 파일과 MyBatis 매핑 XML을 통한 설정 관리
    - Properties 파일: 환경별 설정값 분리 관리
    - Log4j2: 체계적인 로그 관리 및 레벨별 로그 출력 제어

    - 실시간 채팅 시스템 구현 시 동시 접속자 처리 문제와 WebSocket 최적화
    - 기존 시스템과의 호환성 문제와 점진적 마이그레이션 전략
    - 사용자 교육 및 적응 기간 최소화를 위한 직관적 UI/UX 설계
    - 데이터 보안 및 개인정보 보호 강화를 위한 다층 보안 시스템 구축

  2) 업무 효율성 향상

    - 사용자/관리자 활동 로그를 통한 업무 패턴 분석 가능

    - 역할 기반 접근 제어로 체계적인 권한 관리 실현
    - 표준화된 워크플로우로 업무 품질 일관성 확보

    - 직관적 UI/UX로 사용자 학습 부담 최소화

4. 프로젝트를 통해 배우거나 느낀 점

- 박기성: 실제 운영 가능한 웹 시스템을 설계·구현하며 프론트엔드부터 AI 연동까지 풀스택 개발 역량을 키웠다. 기능 구현을 넘어 사용자 중심 설계와 유지보수 구조까지 경험하며 실무 감각과 문제 해결 능력을 향상시켰다.

- 권대용: AI 기반 캡스톤디자인 관리시스템 개발 과정에서 기술 구현뿐 아니라 실제 사용자 경험을 고려한 시스템 설계와 서비스 흐름에 집중할 수 있었다. 특히 사용자 유형별 인터페이스와 접근 권한을 조정하면서, 기술을‘사용자 중심의 문제 해결 도구’로 접근하게 되었다.

- 박현비: AI 기반 시스템을 개발하며 단순한 기능 구현보다, 사용자 입장에서의 흐름과 경험을 더 깊이 고민하게 되었다. 기술이 문제를 해결하는 도구이자 배려의 방식이 될 수 있다는 점을 느낀 값진 경험이었다.

- 천은지: 이번 프로젝트를 통해 프론트엔드와 백엔드를 아우르는 시스템 전반을 이해하고, 기초 설계의 중요성을 체감하였다. 또한 협업과 코드 리뷰를 통해 문제 해결력과 소통 능력을 크게 향상시킬 수 있었다.
