I. 프로젝트 개요
 1. 프로젝트 소개
  본 프로젝트는 IoT 및 AI 기술을 활용하여 냉장고 내부 환경을 실시간으로 모니터링하고, 바코드 기반 식자재 관리, 맞춤형 레시피 추천, 영양 성분 분석 기능을 통합한 지능형 냉장고 관리 시스템을 개발하는 것을 목표로 하였습니다. 현대 사회에서 증가하는 식품 관리의 비효율성과 낭비 문제를 해결하고, 사용자에게 건강한 식습관을 지원하는 스마트 솔루션을 제공하고자 기획되었습니다.

  시스템의 핵심 목표는 사용자가 냉장고를 보다 효율적으로 관리하고, 식재료 낭비를 최소화하며, 영양 균형을 고려한 건강한 식단을 유지할 수 있도록 지원하는 것입니다. 또한 OpenAI API와 데이터 시각화 기술을 활용하여 개인 맞춤형 식단 분석 및 개선 방안을 제안함으로써 사용자의 생활 편의성을 극대화하고자 하였습니다.

 2. 프로젝트 수행 배경 및 필요성
  현대인의 바쁜 생활 속에서 냉장고 관리의 효율성을 높이는 스마트 솔루션에 대한 필요성이 지속적으로 증가하고 있습니다. 많은 가정에서 냉장고 내부 식자재를 효율적으로 관리하지 못해 식품 낭비가 증가하고 있으며, 유통기한이 지난 식재료를 폐기하는 경우가 빈번히 발생하여 비용과 자원 낭비로 이어지고 있습니다.

  기존의 스마트 냉장고는 고가의 제품이 대부분이며, 사용자가 쉽게 접근할 수 있는 자동화된 식자재 관리 및 맞춤형 레시피 추천 기능이 부족한 실정입니다. 또한 냉장고 내부의 온도, 습도, 유해가스 등의 환경을 실시간으로 확인하기 어려워 식품이 최적의 상태에서 보관되지 못하는 경우가 많습니다.

  따라서 IoT 및 AI 기술을 접목한 지능형 냉장고 관리 시스템을 개발함으로써 사용자 편의성을 극대화하고, 식자재 소비 패턴을 분석하여 맞춤형 식단을 추천하며, 건강한 생활을 지원할 수 있는 솔루션의 개발이 필요하다고 판단하였습니다.






 3. 프로젝트 특징
  본 프로젝트는 온습도 센서(DHT11), 유해가스 센서(MQ-135), 바코드 스캐너, AI 기술을 융합한 통합 관리 시스템으로 기존 제품 대비 비용 효율성과 실용성이 우수합니다. 특히 고가의 스마트 냉장고를 구매하지 않고도 기존 냉장고에 IoT 센서를 추가하여 스마트 기능을 구현할 수 있다는 점에서 접근성이 뛰어납니다.

  시스템의 독창성은 단순한 환경 모니터링을 넘어서 OpenAI API를 활용한 지능형 레시피 추천과 영양 성분 분석 기능을 통합하여 사용자의 건강 관리까지 지원한다는 점입니다. 또한 바코드 기반 자동 식자재 등록 시스템과 소비 패턴 분석을 통해 개인 맞춤형 식품 관리 서비스를 제공합니다.

II. 프로젝트 내용
 1. 프로젝트 구성
  지능형 냉장고 관리 시스템은 하드웨어, 소프트웨어의 2계층 구조로 구성됩니다. 하드웨어 계층에서는 Raspberry Pi 5와 Arduino Mega 2560을 중심으로 각종 IoT 센서들이 냉장고 내부 환경 데이터를 수집합니다. 소프트웨어 계층에서는 Node.js 기반의 백엔드 서버와 React.js 기반의 프론트엔드가 데이터 처리와 사용자 인터페이스를 담당합니다.

  시스템 전체 구성도는 IoT 센서 → 마이크로컨트롤러 → 서버 → 데이터베이스 → 웹/모바일 애플리케이션의 순서로 데이터가 흐르며, 각 단계에서 실시간 데이터 처리와 분석이 이루어집니다.

 2. 주요 기능
· 실시간 냉장고 환경 모니터링
  DHT11 온습도 센서, MQ-135 유해가스 센서, 마그네틱 문열림 센서를 활용하여 냉장고 내부 온도, 습도, 공기질 및 문 개폐 상태를 실시간으로 감지하고 기록합니다. 수집된 데이터는 웹 대시보드를 통해 시각화되어 사용자가 냉장고 상태를 한눈에 파악할 수 있습니다.

· 바코드 스캐너 기반 식자재 관리
  바코드 스캐너를 활용하여 식자재 정보를 자동으로 등록하고, 유통기한 및 재고를 실시간으로 관리합니다. 사용자는 식품을 구입할 때 바코드를 스캔하면 자동으로 시스템에 등록되며, 유통기한 임박 시 자동 알림을 받을 수 있습니다.


· AI 기반 맞춤형 레시피 추천
  OpenAI API를 활용하여 냉장고에 보관 중인 식자재를 기반으로 맞춤형 레시피를 추천합니다. 사용자의 취향, 알레르기 정보, 영양 선호도를 고려하여 개인화된 요리법을 제안하며, 재료 소진을 위한 효율적인 레시피를 우선적으로 추천합니다.

· 식자재 소비 패턴 분석
  사용자의 식자재 소비 데이터를 분석하여 소비 패턴 통계를 제공하고, 효율적인 식자재 소비 방안을 제안합니다. Chart.js를 활용한 데이터 시각화를 통해 월별, 주별 소비 패턴을 그래프로 표시하며, 낭비 요소를 식별하여 개선 방안을 제시합니다.

· 영양 성분 분석 및 건강 관리
  사용자 식단의 영양 성분을 분석하여 균형 잡힌 식단을 제안하고 건강 관리 솔루션을 제공합니다. 일일 권장 영양소 섭취량과 비교하여 부족하거나 과다한 영양소를 식별하고, 이를 보완할 수 있는 식품을 추천합니다.

· 유통기한 관리 및 사용자 알림
  유통기한 임박 식자재를 자동으로 감지하여 알림을 제공하고, 효율적인 소비를 위한 레시피 추천과 연계합니다. 모바일 푸시 알림, 웹 알림, 이메일 알림을 통해 다양한 채널로 사용자에게 정보를 전달합니다.

 3. 주요 기술
· 백엔드 기술
  Node.js와 Express.js를 활용하여 웹 애플리케이션의 백엔드를 구현하였습니다. RESTful API 설계를 통해 프론트엔드, IoT 기기 간의 효율적인 데이터 통신을 구현하였습니다.

· 데이터베이스 관리
  MySQL(MariaDB)을 활용하여 식자재 정보, 냉장고 상태 데이터를 체계적으로 저장하고 관리합니다. 데이터베이스 정규화를 통해 데이터의 일관성을 유지하고, 인덱싱을 통해 조회 성능을 최적화하였습니다.

· 프론트엔드 기술
  React.js를 활용하여 동적인 사용자 인터페이스를 구현하고 반응형 웹 디자인을 적용하였습니다. 사용자 경험을 최적화하기 위해 직관적인 UI/UX 디자인을 적용하였으며, 모바일과 데스크톱 환경에서 모두 원활하게 작동하도록 개발하였습니다.


· AI 및 머신러닝
  OpenAI API를 활용하여 사용자의 냉장고 내 보관 중인 식자재를 분석하고, 맞춤형 레시피 및 영양 분석을 제공하는 AI 기반 서비스를 구현하였습니다. 자연어 처리를 통해 사용자 질의에 대한 정확한 응답을 생성하고, 개인화된 추천 시스템을 구축하였습니다.

· IoT 기술
  Raspberry Pi 5와 Arduino Mega 2560을 활용하여 IoT 센서 데이터를 수집하고 냉장고 환경 모니터링 시스템을 구축하였습니다. 센서 데이터의 실시간 수집과 전송을 위해 MQTT 프로토콜을 사용하였습니다.

 3. 프로젝트 도전
· OpenAI API 응답 지연 문제
  레시피 추천 기능에서 OpenAI API 호출 시 응답 시간이 길어져 사용자 경험이 저하되는 문제가 발생하였습니다. 이를 해결하기 위해 비동기 처리를 통한 논블로킹 방식으로 API 호출을 개선하고, 자주 사용되는 레시피는 캐싱 시스템을 도입하여 응답 속도를 50% 이상 향상시켰습니다.

· 데이터베이스 성능 최적화
  대량의 센서 데이터가 실시간으로 저장되면서 데이터베이스 조회 성능이 저하되는 문제가 발생하였습니다. 인덱싱 최적화, 파티셔닝 적용, 그리고 불필요한 데이터의 주기적 정리를 통해 데이터베이스 성능을 개선하였으며, 조회 속도를 약 3배 향상시켰습니다.

· 바코드 인식 정확도 문제
  바코드 스캐너의 인식률이 조명 조건과 각도에 따라 불안정한 문제가 있었습니다. 이미지 전처리 알고리즘을 적용하고, 다양한 각도에서의 스캔을 지원하는 기능을 추가하여 바코드 인식률을 95% 이상으로 향상시켰습니다.

4. 프로젝트를 통해 배우거나 느낀 점
· 팀장 한승현
  프로젝트를 총괄하면서 팀 관리와 기술적 이슈 해결을 동시에 진행하는 것의 어려움을 깨달았습니다. 특히 IoT 하드웨어와 소프트웨어를 통합하는 과정에서 예상치 못한 호환성 문제들을 해결하며 시스템 통합의 복잡성을 이해하게 되었습니다. 또한 팀원들과의 소통과 일정 관리의 중요성을 실감하였으며, 리더십과 기술적 역량을 동시에 기를 수 있는 소중한 경험이었습니다.

· 팀원1 박지민
  AI 기술을 실제 서비스에 적용하는 과정에서 이론과 실무의 차이를 체감할 수 있었습니다. OpenAI API 연동과 데이터 분석 알고리즘 구현을 통해 실무에서 활용할 수 있는 AI 개발 역량을 기를 수 있었습니다. 특히 사용자 데이터를 분석하여 의미 있는 인사이트를 도출하는 과정에서 데이터 사이언스의 실무적 접근 방법을 배울 수 있었습니다.

· 팀원2 박지현
  사용자 중심의 UI/UX 설계가 얼마나 중요한지 깨달았습니다. 복잡한 데이터를 직관적으로 시각화하는 과정에서 사용자의 관점에서 생각하는 능력이 향상되었으며, React.js를 활용한 컴포넌트 기반 개발 방식을 익힐 수 있었습니다. 또한 반응형 웹 디자인을 통해 다양한 디바이스에서의 사용성을 고려하는 중요성을 이해하게 되었습니다.

· 팀원3 정유선
  백엔드 개발을 담당하면서 확장 가능한 서버 아키텍처 설계의 중요성을 깨달았습니다. 실시간 데이터 처리와 API 성능 최적화 과정에서 많은 시행착오를 겪었지만, 이를 통해 실무에서 요구되는 백엔드 개발 역량을 크게 향상시킬 수 있었습니다. 특히 AWS 클라우드 서비스 활용 경험을 통해 실제 서비스 운영에 필요한 인프라 지식을 습득하게 되었습니다.
